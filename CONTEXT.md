# XNOWPost — 领域语言与架构上下文

> 内容营销自动化引擎。自动化内容采集、AI 生成、多平台发布。

---

## 核心概念

### 内容 / Content
一篇文章或帖子的原始素材，可能来自 RSS 采集、AI 生成、或手动创建。

### 发布平台 / Platform
目标发布渠道，如微信公众号、知乎、小红书、Twitter 等。每个平台有不同的格式要求和发布 API。

### 采集器 / Collector
从 RSS、API 或其他源自动抓取内容的模块。
_Avoid_: 爬虫、抓取器

### 渲染器 / Renderer
将内容转换为目标平台格式的模块（如 Markdown → 微信公众号格式）。
_Avoid_: 转换器

### 引擎 / Engine
核心调度模块，管理采集 → 处理 → 渲染 → 发布的完整流水线。

### 分析器 / Analyzer
对已发布内容进行效果分析（阅读量、互动等）的模块。

---

## 架构决策 (ADRs)

### ADR-001: 流水线架构
**决策：** 采集 → 处理 → 渲染 → 发布，每个阶段独立模块，通过队列连接。
**原因：** 各阶段可以独立开发、测试、替换，某阶段失败不影响其他阶段。

## 目录结构

```
src/
├── collector/     # 内容采集
├── engine/        # 核心调度
├── platforms/     # 各平台发布实现
├── renderer/      # 内容格式转换
├── analyzer/      # 效果分析
├── organizer/     # 内容管理
├── notifier.js    # 通知
└── scheduler.js   # 定时调度
```
